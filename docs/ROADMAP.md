# Rebuild Roadmap

## Phase 1 — Product Foundation
- tenant-first Supabase schema
- organization-scoped memberships and roles
- encrypted connector configuration model
- architecture decision records
- clean app shell and runtime health route

## Phase 2 — Adapter and Sync Spine
- Notion CRM adapter
- Nabis orders adapter
- sync cursor + sync job orchestration
- deterministic account identity and reconciliation queue
- audit/event foundation

## Phase 3 — Unified Territory Runtime
- local territory pin read model
- shared territory boundaries and markers
- Google Maps rendering and routing integration
- targeted invalidation instead of aggressive polling

## Phase 4 — Unified Account System
- account detail read model
- contact/activity model
- local order-derived aggregates
- outbound CRM sync for display fields

## Phase 5 — Calendar and Operations
- local calendar event model
- Google Calendar adapter
- vendor-day operational workflow
- organization-aware notifications and activity views

## Phase 6 — Productization
- tenant admin UI for connectors and field mappings
- onboarding flow for a new company
- fake sandbox tenant for demos
- performance and observability dashboards
