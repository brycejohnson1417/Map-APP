## 2024-06-25 - Workspace Experience Optimization
**Learning:** React's `cache` is used for `getWorkspaceExperienceBySlug` which performs a database call (`organizations.findBySlug`). When building UI components or processing logic where the `Organization` object is already available (e.g. from `tenant-access.ts` resolving from memberships list), calling `getWorkspaceExperienceBySlug(organization.slug)` creates a redundant database query because it bypasses the locally available data and uses only the slug as the cache key.
**Action:** When the `Organization` object is already fetched and available, bypass `getWorkspaceExperienceBySlug` and instead call `compileWorkspaceExperience({ slug: organization.slug, organization })` directly to construct the workspace experience synchronously and avoid unnecessary database queries. Make sure to provide the full `Organization` object as `compileWorkspaceExperience` expects `Pick<Organization, 'slug' | 'settings'>`.
