## 2025-06-13 - Eliminate N+1 Database Lookup in cache() Methods
**Learning:** In Next.js, `React.cache()` creates separate cache entries if the arguments differ by object reference. When trying to optimize redundant database lookups by passing an optional `Organization` object down the call stack, doing so on a `cache()` wrapped function fragments the cache instead of improving it, which executes the inner synchronous logic multiple times instead.
**Action:** When implementing database query bypass mechanisms for performance, either ensure the `cache()` wrapper focuses solely on the database query (e.g. `cache((slug) => db.find(slug))`) instead of the entire function, or ensure that you are fully aware of how React cache handles new object arguments to avoid defeating the memoization.
