## 2024-05-14 - Map Component Array Search Bottleneck
**Learning:** Found a common React performance bottleneck in `components/territory/territory-workspace.tsx` where an array was being mapped over (`routeStopIds.map`), and for each element, a nested array search (`pins.find`) was performed to resolve the ID to an object. This pattern creates an O(N * M) time complexity directly inside a component render pass (or `useMemo`), which blocks the main thread when data arrays grow.
**Action:** Always refactor nested `.map` / `.find` array lookups inside React render cycles into a one-time O(N) `Map` initialization (e.g., `useMemo(() => new Map(data.map(item => [item.id, item])), [data])`), followed by O(1) `.get()` lookups. This pattern should be standard for data resolution across all complex frontend UI components.
