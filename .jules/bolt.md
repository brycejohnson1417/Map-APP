## 2024-10-24 - Avoid redundant DB queries on synchronous compile paths
**Learning:** In the `tenant-access.ts` file, the `resolveTenantAccess` function initially fetched an `Organization` object. Instead of directly using that fetched `Organization` object with the synchronous `compileWorkspaceExperience` compiler from `lib/platform/workspace/compiler`, the code made another redundant asynchronous lookup via `getWorkspaceExperienceBySlug(organization.slug)`, which internally repeated the lookup of the `Organization` using `organizations.findBySlug(slug)`.
**Action:** When a method needs to compile a workspace definition and the `Organization` object has already been fetched, bypass the asynchronous service method `getWorkspaceExperienceBySlug` and explicitly call the synchronous `compileWorkspaceExperience` with the fetched `Organization` object to avoid an N+1 query problem on critical request paths. Note: The `CompiledWorkspaceExperience` interface returned does not contain an `.organization` property.
## 2024-10-24 - Ensure full object properties when casting/typing for compilers
**Learning:** When using `compileWorkspaceExperience` directly to skip redundant DB lookups, it requires `settings` on the `Organization` object (`Pick<Organization, "slug" | "settings">`). The fallback type `MembershipSelectionOrganization` didn't have this property, which led to a type mismatch error when removing an `as any` type assertion.
**Action:** Always ensure that intermediary projection interfaces (like `MembershipSelectionOrganization`) include the required properties needed by downstream compilers instead of using `as any`, which subverts safety checks and risks runtime crash.
## 2024-10-24 - Maintain domain boundaries when optimizing layer communications
**Learning:** In attempting to eliminate a redundant DB fetch, directly calling `compileWorkspaceExperience` inside `resolveTenantAccess` bypassed the service layer (`getWorkspaceExperienceBySlug`) and leaked internal compiler requirements (like the `settings` property on the `Organization` object) into the application auth layer. This caused type errors and broke encapsulation.
**Action:** When a service method makes a redundant lookup for an object the caller already has, the proper optimization is to update the service method signature to accept an optional pre-fetched object (e.g., `prefetchedOrganization?: Organization | null`). This eliminates the N+1 query problem while preserving clean domain boundaries and type safety.
