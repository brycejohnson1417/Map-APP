## 2024-05-14 - Replace redundant `getWorkspaceExperienceBySlug` calls
**Learning:** `getWorkspaceExperienceBySlug(slug)` queries the DB for an organization and then calls `compileWorkspaceExperience`. In many places (like `lib/application/auth/tenant-access.ts`), we already fetched the `organization` object from the database, but we still pass `organization.slug` into `getWorkspaceExperienceBySlug`, causing an unnecessary redundant DB lookup for the organization!
**Action:** Replace `await getWorkspaceExperienceBySlug(organization.slug)` with `compileWorkspaceExperience({ slug: organization.slug, organization })` whenever the `organization` is already available. Note: `compileWorkspaceExperience` returns `CompiledWorkspaceExperience` which does NOT have an `.organization` property, so we must manually pass it if needed, or simply access properties directly from the `CompiledWorkspaceExperience`.
