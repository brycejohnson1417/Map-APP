## 2025-01-20 - Memoize O(N) Array Finds in territory-workspace.tsx
**Learning:** O(N) array finds (`Array.prototype.find`) inside `useMemo` block the main thread and can become significant performance bottlenecks when scaling to thousands of pins (e.g. `pins.find` inside `routeStopIds.map` results in O(N*M) complexity). React's rendering lifecycle slows down when these hooks depend on large lists.
**Action:** Replaced linear array searches inside `useMemo` loops with an O(1) memoized Map (`pinsById`). This guarantees predictable O(1) performance lookup regardless of map scale. However, specifically preserved the `find()` calls inside `window.__MAP_APP_TEST` global hooks, since external imperative bindings may have different variable scopes, which avoids stale closure bugs.
