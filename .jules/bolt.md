## 2025-02-28 - [Territory Pins Optimization]
**Learning:** External test bindings that capture functions interacting with React-managed objects (e.g., `window.__MAP_APP_TEST.getLeafletClickPointForPin`) shouldn't use React memoized maps. If we had replaced the `pins.find` in the global `__MAP_APP_TEST` assignment with `pinsById.get`, the external test harness could experience stale closures when the React component renders and updates the `pinsById` Map without recreating the external binding.
**Action:** When memoizing maps inside components for internal rendering performance, only use the Map for internal render/effect state dependencies. Retain standard array methods (like `.find()`) inside external, globally-assigned testing bounds (like `window.__MAP_APP_TEST`) unless you also update how they synchronize with the React component lifecycle.
