## 2024-03-24 - Avoid redundant database queries for compiled workspace experiences
**Learning:** Performance Optimization Pattern: Avoid redundant database queries by passing already-fetched `Organization` objects directly to synchronous compilers (e.g., `compileWorkspaceExperience` from `@/lib/platform/workspace/compiler`) instead of calling async services like `getWorkspaceExperienceBySlug` that perform a redundant `findBySlug` lookup. Note that `CompiledWorkspaceExperience` does not include the `organization` property in its return type.
**Action:** When a method already fetches an `Organization` or has access to it, pass it directly to `compileWorkspaceExperience({ slug: organization.slug, organization })` rather than passing the slug to `getWorkspaceExperienceBySlug(slug)`.
