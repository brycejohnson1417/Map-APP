## 2026-06-08 - Use Synchronous Compilers for Fetched Objects
**Learning:** Avoid redundant database queries by passing already-fetched 'Organization' objects directly to synchronous compilers (e.g., 'compileWorkspaceExperience' from '@/lib/platform/workspace/compiler') instead of calling async services like 'getWorkspaceExperienceBySlug' that perform a redundant 'findBySlug' lookup. Also, note that 'CompiledWorkspaceExperience' does not include the 'organization' property in its return type.
**Action:** Before calling service layer functions that fetch data (like `getWorkspaceExperienceBySlug`), check if you already have the required domain objects in scope. If you do, use the underlying synchronous compilation/mapper functions directly (like `compileWorkspaceExperience`) instead of making a round-trip to the service layer.
