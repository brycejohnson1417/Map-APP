## 2024-05-18 - Prevent Main Thread Blocking in React Memos with Map Lookups
**Learning:** Performing `Array.find` inside an `Array.map` within a React `useMemo` hook (like mapping route stop IDs to their pin objects via `pins.find(pin => pin.id === id)`) results in `O(N*M)` time complexity. In components with many items (e.g. territory workspace maps with hundreds or thousands of pins), this synchronous operation can block the main thread and severely degrade React render performance and UI responsiveness.
**Action:** Always construct a dictionary (`Map`) beforehand to convert these nested loops into `O(1)` map lookups (`pinsMap.get(id)`), ensuring the complexity becomes `O(N+M)`. When implementing this in React components, ensure the dictionary itself is also correctly memoized (`useMemo`) to prevent unnecessary recreation.
