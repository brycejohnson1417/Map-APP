## 2024-06-25 - Prevent XSS in Email Previews Using Sandboxed Iframes
**Vulnerability:** A cross-site scripting (XSS) vulnerability was found where `dangerouslySetInnerHTML` was used in conjunction with `DOMPurify` to render email previews. Even with `DOMPurify`, rendering untrusted HTML directly into the DOM can be risky if purification bypasses are discovered or if the configuration is flawed.
**Learning:** Using a sandboxed `iframe` with the `srcDoc` attribute is a far safer mechanism for rendering untrusted or dynamic HTML like email previews. The `sandbox` attribute provides a deeply restricted, isolated browsing context that intrinsically prevents script execution, without relying entirely on a userland purification step.
**Prevention:** Always use `iframe` with `srcDoc` and `sandbox="allow-popups allow-popups-to-escape-sandbox"` (which allows clicking links safely without executing scripts) when rendering raw email HTML or other untrusted web content. Avoid `dangerouslySetInnerHTML` for this purpose.
