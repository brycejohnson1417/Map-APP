## 2025-02-14 - Replace dangerouslySetInnerHTML with Sandboxed iframe for Untrusted HTML Preview
**Vulnerability:** XSS vulnerability through untrusted HTML strings passed to `dangerouslySetInnerHTML`. Even when using `DOMPurify`, sophisticated attacks or misconfigurations can lead to XSS.
**Learning:** For rendering completely standalone HTML like email previews, `dangerouslySetInnerHTML` is unnecessary and risky. Using an `<iframe srcDoc="...">` with the `sandbox` attribute restricts the context entirely, preventing scripts from executing while still rendering HTML properly. Adding `sandbox="allow-popups allow-popups-to-escape-sandbox"` permits links to open safely without allowing scripts or same-origin access.
**Prevention:** Avoid `dangerouslySetInnerHTML` entirely for rendering full HTML documents like email templates or user-submitted HTML blocks. Default to using sandboxed iframes to enforce a strict security boundary, treating the HTML as completely untrusted content.
