## 2024-05-24 - Secure Email Previews with Sandboxed Iframes
**Vulnerability:** A cross-site scripting (XSS) vulnerability pattern was present where `dangerouslySetInnerHTML` was used to render sanitized HTML for email previews (`components/accounts/ppp-savings-panel.tsx`). While `DOMPurify` was used, it may not be sufficient on its own.
**Learning:** Even with server-side or client-side sanitization, rendering untrusted or dynamic HTML directly into the React DOM using `dangerouslySetInnerHTML` exposes the application to potential XSS vulnerabilities, especially if the sanitization configuration is bypassed or flawed.
**Prevention:** Always prioritize using a sandboxed `iframe` with the `srcDoc` attribute and a strict `sandbox=""` attribute when rendering untrusted or dynamic HTML (like email previews). This creates a strong, isolated browsing context that prevents scripts from executing and interacting with the parent application's DOM, providing a defense-in-depth security layer.
