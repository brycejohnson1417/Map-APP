## 2024-06-08 - Use sandboxed iframes for untrusted HTML instead of dangerouslySetInnerHTML
**Vulnerability:** XSS vulnerability potential when rendering untrusted or dynamic email HTML in the UI via `dangerouslySetInnerHTML`.
**Learning:** Even when sanitized (e.g. `DOMPurify.sanitize`), `dangerouslySetInnerHTML` directly injects nodes into the parent DOM, which can lead to layout disruption and style leakage. In addition, relying purely on sanitization exposes us to XSS if the sanitizer has a zero-day bypass. The safest way to render fully dynamic or external HTML content (like email previews) is in a strict browsing context isolation boundary.
**Prevention:** Always use a sandboxed `iframe` with `srcDoc` and explicitly restrictive `sandbox` attributes (e.g., `sandbox="allow-popups allow-popups-to-escape-sandbox"` to only allow external link clicking while strictly omitting `allow-scripts` and `allow-same-origin`) when displaying HTML previews, instead of using `dangerouslySetInnerHTML`.
