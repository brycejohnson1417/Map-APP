## 2025-06-16 - Prevent XSS in Email Previews
**Vulnerability:** Use of `dangerouslySetInnerHTML` for rendering dynamic email previews. Even with DOMPurify, there is a risk of XSS or layout breaking if the HTML content interacts unexpectedly with the main DOM.
**Learning:** For rendering untrusted or dynamic HTML (like email previews), the standard security approach should involve a defense-in-depth strategy. An iframe creates an isolated browsing context that protects the parent document, allowing you to explicitly restrict execution using the `sandbox` attribute (e.g. only allowing links to be clicked with `allow-popups allow-popups-to-escape-sandbox`). Furthermore, inline `<style>` injection inside the iframe's `srcDoc` is required because outer CSS (like Tailwind classes) do not affect inner documents.
**Prevention:** Always default to sandboxed `<iframe srcDoc={...}>` instead of `dangerouslySetInnerHTML` when displaying email or external HTML content. Avoid using `allow-scripts` or `allow-same-origin` in the sandbox attributes.
