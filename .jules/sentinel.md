## 2024-06-25 - Fix XSS vulnerability by replacing dangerouslySetInnerHTML with sandboxed iframe
**Vulnerability:** XSS vulnerability in email preview rendering via `dangerouslySetInnerHTML`.
**Learning:** Even with DOMPurify sanitization, rendering untrusted or dynamic HTML using `dangerouslySetInnerHTML` directly in the DOM structure can pose a risk, especially if sanitization misses anything. A sandboxed iframe provides a more robust, browser-level isolation layer for rendering third-party or dynamic HTML content safely.
**Prevention:** Always prioritize using a sandboxed `iframe` with the `srcDoc` attribute instead of `dangerouslySetInnerHTML` when rendering dynamic or untrusted HTML content. Include `sandbox="allow-popups allow-popups-to-escape-sandbox"` to permit links while blocking scripts and same-origin access. Inject inline `<style>` tags directly into the `srcDoc` string to apply styling, since iframes create an isolated browsing context that won't inherit parent CSS classes (like Tailwind).
